# 编译构建指南

本文档介绍如何使用命令行编译和构建IntelliJ IDEA日报周报插件。

## 前置要求

- JDK 11 或更高版本
- Gradle 8.5（项目已包含）
- 网络连接（用于下载依赖）

## 项目结构

```
daily-report-plugin/
├── build.gradle.kts               # Gradle构建配置
├── gradle-8.5/                 # Gradle运行时（便携版）
├── gradlew.bat                  # Gradle包装脚本（Windows）
├── src/                        # 源代码
└── build/                       # 构建输出
```

## 编译步骤

### 1. 清理构建

```bash
gradle-8.5\bin\gradle.bat clean
```

### 2. 编译源代码

```bash
gradle-8.5\bin\gradle.bat compileJava
```

### 3. 运行测试

```bash
gradle-8.5\bin\gradle.bat test
```

### 4. 构建插件

```bash
gradle-8.5\bin\gradle.bat build
```

## 构建产物

构建成功后，会在以下目录生成文件：

### 插件JAR包

```
build/libs/daily-report-plugin-1.0.0.jar
```

这是主要的插件文件，可以直接安装到IntelliJ IDEA中。

### 插件分发包

```
build/distributions/daily-report-plugin-1.0.0.zip
```

包含插件及其所有依赖的压缩包。

## 常见问题

### 1. 编译时出现GBK编码错误

**问题**：编译时提示"编码 GBK 的不可映射字符"

**解决**：项目已配置UTF-8编码，确保在支持UTF-8的环境中编译。

### 2. 依赖下载失败

**问题**：无法下载依赖包

**解决**：
- 检查网络连接
- 配置国内镜像源（如阿里云Maven镜像）
- 清理Gradle缓存：`gradle-8.5\bin\gradle.bat clean --no-daemon`

### 3. IntelliJ SDK相关错误

**问题**：提示IntelliJ Platform SDK相关错误

**解决**：
- 确保JDK版本正确（JDK 11+）
- 清理IDEA缓存：`File` -> `Invalidate Caches / Restart...`

### 4. 构建警告

**问题**：显示"Java configuration specifies sourceCompatibility=11 but IntelliJ Platform 2022.3 requires sourceCompatibility=17"

**说明**：这是配置警告，不影响构建。插件仍然可以正常工作。

## 开发调试

### 本地测试插件

1. 构建插件：`gradle-8.5\bin\gradle.bat build`
2. 在IDEA中安装插件：`File` -> `Settings` -> `Plugins` -> `Install Plugin from Disk...`
3. 选择生成的JAR文件
4. 重启IDEA

### 运行测试类

项目包含测试类用于验证功能：

#### AI连接测试

```bash
java -cp "build/classes/java/main;build/classes/java/test;build/idea-sandbox/plugins/daily-report-plugin/lib/*" com.dailyreport.test.AITest
```

#### 功能测试

```bash
java -cp "build/classes/java/main;build/classes/java/test;build/idea-sandbox/plugins/daily-report-plugin/lib/*" com.dailyreport.test.FunctionTest
```

## Gradle命令参考

### 常用命令

| 命令 | 说明 |
|-------|------|
| `clean` | 清理构建输出 |
| `compileJava` | 编译主代码 |
| `compileTestJava` | 编译测试代码 |
| `test` | 运行测试 |
| `build` | 完整构建（包括测试） |
| `buildPlugin` | 仅构建插件JAR |
| `prepareSandbox` | 准备IDEA沙箱环境 |

### 组合使用

```bash
# 清理并重新构建
gradle-8.5\bin\gradle.bat clean build

# 仅编译不运行测试
gradle-8.5\bin\gradle.bat compileJava compileTestJava

# 构建并跳过测试
gradle-8.5\bin\gradle.bat build -x test
```

## 依赖管理

### 查看依赖树

```bash
gradle-8.5\bin\gradle.bat dependencies
```

### 更新依赖

```bash
gradle-8.5\bin\gradle.bat build --refresh-dependencies
```

## 发布插件

构建完成后，可以将插件发布到IntelliJ Plugin Marketplace：

1. 注册JetBrains账号：https://plugins.jetbrains.com/
2. 上传插件JAR文件
3. 填写插件信息和文档
4. 等待审核

## 技术支持

如遇到构建问题，请检查：
1. JDK版本是否正确
2. 网络连接是否正常
3. Gradle版本是否匹配
4. 项目依赖是否完整

更多帮助请参考：
- IntelliJ Platform SDK文档：https://plugins.jetbrains.com/docs/intellij/welcome.html
- Gradle文档：https://docs.gradle.org/
