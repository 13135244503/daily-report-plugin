### 需求背景
我需要开发一个 IntelliJ IDEA 插件（Maven 构建），核心功能是自动生成多项目维度的工作日报/周报，集成百度千帆 Coding Plan 专属 AI 接口分析工作内容，具体要求如下：

### 核心功能清单
1. **多项目 Git 日志解析**
    - 支持配置多个项目目录（本地代码路径），插件会遍历所有配置的「启用状态」项目目录
    - 针对每个配置项目，按选择的时间维度（日报/周报）解析 Git 日志
    - 汇总所有项目的 Git 记录（commit 信息、代码改动、文件列表等），按项目维度分类展示
    - 分析未提交的改动，包括修改、新增、删除和未跟踪的文件
    - Git 仓库不存在/目录无效时自动跳过，报告中备注异常项目

2. **灵活的时间维度筛选**
    - 日报：筛选所选日期（默认当天）00:00 至 23:59 的 Git 日志
    - 周报：筛选本周一 00:00 至本周日 23:59 的 Git 日志（支持自定义一周起始日）
    - 支持手动选择时间范围，补生成历史日报/周报

3. **全维度自定义配置（重点优化 Coding Plan 接口配置）**
    - 插件配置界面（Settings/Preferences → 日报周报插件）分为 3 个子面板：
      ① 项目配置：添加/删除/启用/禁用项目目录，支持「浏览」选择本地路径
      ② 输出配置：自定义报告输出根路径，支持「浏览」选择，默认：用户目录/工作报告/
      ③ AI 配置（适配 Coding Plan 专属接口）：
        - 接口协议选择：下拉框选择「OpenAI 协议」/「Anthropic 协议」
        - Base URL 输入框：根据协议默认填充对应 Coding Plan 专属 Base URL，支持手动修改
            - OpenAI 协议默认值：https://qianfan.baidubce.com/v2/coding
            - Anthropic 协议默认值：https://qianfan.baidubce.com/anthropic/coding
        - API Key 输入框：脱敏显示（输入后显示 ****），必填项
        - 请求超时时间：输入框（默认 10 秒）
        - 测试连接按钮：验证 Base URL + API Key 是否有效
    - 所有配置持久化存储（IDE 重启后不丢失），支持「恢复默认配置」

4. **Coding Plan AI 内容分析（核心适配）**
    - 根据选择的接口协议，自动拼接完整接口路径：
        - OpenAI 协议完整路径：{Base URL}/chat/completions
        - Anthropic 协议完整路径：{Base URL}/v1/messages
    - 将多项目 Git 日志汇总为结构化数据，按对应协议格式封装请求体
    - 调用 Coding Plan 接口生成工作总结，失败时降级为本地总结

5. **多项目报告生成**
    - 触发方式：IDEA 顶部菜单/右键菜单 → 「生成工作报告」→ 弹窗选择「日报」/「周报」
    - 文件命名规则：
        - 日报：输出路径/YYYY-MM-DD-多项目工作日报.md
        - 周报：输出路径/YYYY年第WW周-多项目工作周报.md
    - 报告按项目维度分类展示，包含 AI 生成的工作总结

### 技术要求
1. 开发环境：JDK 11 + Maven + IntelliJ IDEA SDK（2022.3+）
2. 核心依赖：
    - JGit（解析 Git 日志）、commons-lang3（日期/字符串工具）
    - okhttp（AI 接口调用）、gson（JSON 解析）
    - IDEA SDK 核心组件（PersistentStateComponent、Configurable、FileChooser 等）
3. 兼容版本：IntelliJ IDEA 社区版/旗舰版 2022.3+
4. 异常处理：
    - 配置的项目目录不存在/非 Git 仓库：跳过该项目，日志记录并在报告中备注
    - AI 接口调用失败/超时：降级为本地总结，弹窗提示原因
    - 输出路径无写入权限：弹窗提示并引导重新选择路径

### 交付物要求
1. 完整的 Maven 项目代码（pom.xml + 所有 Java 类 + plugin.xml + 资源文件）
2. 核心类说明：
    - ProjectConfig：单个项目配置（路径、是否启用）
    - PluginGlobalConfig：全局配置（多项目列表、输出路径、AI 配置<含协议/BaseURL/APIKey/超时>）
    - PluginConfigComponent：配置持久化（PersistentStateComponent）
    - PluginConfigurable：配置界面（重点实现 AI 配置面板的协议切换 + BaseURL 自动填充）
    - MultiProjectGitAnalyzer：多项目 Git 日志解析器
    - CodingPlanAIAnalyzer：适配 Coding Plan 接口的 AI 分析器（区分 OpenAI/Anthropic 协议）
    - MultiProjectReportGenerator：多项目报告生成器
    - GenerateReportAction：触发类，执行生成逻辑
3. 插件打包说明（Maven 命令）、安装教程、使用示例（含 Coding Plan 配置截图）

### 关键细节（重点适配 Coding Plan 接口）
1. Coding Plan 接口配置逻辑：
    - 协议下拉框切换时，自动填充对应默认 Base URL，保留手动修改能力
    - OpenAI 协议：
        - 默认 Base URL：https://qianfan.baidubce.com/v2/coding
        - 完整接口路径：{Base URL}/chat/completions
        - 请求体格式（兼容 OpenAI）：
          {
          "model": "coding-plan",
          "messages": [{"role": "user", "content": "{promptContent}"}],
          "apiKey": "{apiKey}",
          "temperature": 0.2,
          "max_tokens": 1000
          }
    - Anthropic 协议：
        - 默认 Base URL：https://qianfan.baidubce.com/anthropic/coding
        - 完整接口路径：{Base URL}/v1/messages
        - 请求体格式（兼容 Anthropic）：
          {
          "model": "coding-plan",
          "messages": [{"role": "user", "content": "{promptContent}"}],
          "apiKey": "{apiKey}",
          "temperature": 0.2,
          "max_tokens": 1000
          }
    - API Key 为 Coding Plan 专属额度密钥，输入框脱敏显示
    - 「测试连接」按钮逻辑：调用对应协议的接口发送测试 Prompt，验证 Base URL + API Key 有效性

2. 多项目 Git 解析逻辑：
    - 遍历配置的所有「启用」项目目录，检查 .git 文件夹是否存在
    - 汇总数据结构：Map<String, GitStats>（key=项目名称/路径，value=该项目 Git 统计）

3. 标准化 Prompt 模板（适配 Coding Plan）：
   "请基于以下多个项目的 Git 提交记录和代码改动（包括未提交的改动），总结{报告类型}的工作内容，要求：
    1. 按项目维度分点总结核心工作（功能开发/BUG 修复/优化等）
    2. 突出每个项目的技术关键点、解决的问题和工作成果
    3. 语言简洁专业，符合职场工作报告规范，避免冗余
       多项目 Git 记录：
       {multiProjectGitData}"
       （{报告类型} 替换为「今日」/「本周」，{multiProjectGitData} 替换为结构化 Git 日志）

4. 多项目报告模板：
   # 多项目工作{报告类型}
   ## 基本信息
    - 时间范围：{开始时间} 至 {结束时间}
    - 统计项目数：{有效项目数}（共配置 {总项目数} 个，{无效项目数} 个非 Git 项目/禁用）
    - 总提交数：{所有项目提交数总和}
    - 总新增代码行数：{总和} | 总删除代码行数：{总和} | 总净增行数：{总和}

   ## 各项目详情
   {按项目循环渲染：
   ### 项目：{项目名称}（{项目路径}）
    - 提交数：{该项目提交数}
    - 新增行数：{该项目新增} | 删除行数：{该项目删除} | 净增行数：{该项目净增}
    - 文件类型分布：{该项目文件类型}
    - 修改文件列表：{该项目文件列表}
    - Git 提交记录：{该项目提交记录}
    - 未提交的改动：{该项目未提交改动}
      }

   ## 工作内容总结（Coding Plan AI 生成）
   {AI 生成的多项目总结，按项目分点}

   ## 整体总结
   {AI 生成的整体工作成果总结}
